# 자격증명 관리

외부 서비스(위하고, 홈택스 등)의 로그인 자격증명을 Vault에 안전하게 저장하고 관리합니다.
저장된 자격증명은 [세션 생성(로그인)](/api-reference/sessions)에서 `credential_id`로 참조하여 사용합니다.

## 흐름

```
1. 자격증명 등록              POST /api/v1/credentials
2. 세션 생성 (로그인)         POST /api/v1/sessions/{site}/login  ← credential_id 전달
3. 스크래핑                   POST /api/v1/scraping/{site}/{endpoint}  ← session_id 전달
```

<Note>
  자격증명은 **비밀값(secrets)**과 **메타데이터**로 분리 저장됩니다.
  조회 API는 메타데이터만 반환하며, 비밀값은 절대 노출되지 않습니다.
</Note>

---

## 자격증명 생성

```bash
POST /api/v1/credentials
Content-Type: application/json
Authorization: Bearer {keyId}.{secret}
```

```json
{
  "site": "wehago",
  "login_type": "password",
  "secrets": {
    "portal_id": "user@wehago.com",
    "password": "password123"
  }
}
```

### 요청 필드

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `site` | string | O | 서비스 코드 (`wehago`, `hometax`) |
| `login_type` | string | O | 로그인 방식 (`password`, `cert`) |
| `secrets` | object | O | 로그인 비밀값 (서비스·방식별 상이) |

### 서비스별 `secrets`

<Tabs>
  <Tab title="위하고 (password)">
| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `portal_id` | string | O | 위하고 포털 ID |
| `password` | string | O | 비밀번호 |

```json
{
  "site": "wehago",
  "login_type": "password",
  "secrets": {
    "portal_id": "myaccount",
    "password": "mypassword"
  }
}
```
  </Tab>
  <Tab title="홈택스 (cert)">
| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `cert_data` | string | O | Base64 인코딩된 인증서 (DER) |
| `cert_password` | string | O | 인증서 비밀번호 |

```json
{
  "site": "hometax",
  "login_type": "cert",
  "secrets": {
    "cert_data": "base64-encoded-cert",
    "cert_password": "certpassword"
  }
}
```
  </Tab>
</Tabs>

### 응답 — `200 OK`

```json
{
  "credential_id": "cred_57ea61f05c79",
  "verification_status": "unverified"
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `credential_id` | string | 자격증명 ID (`cred_` 접두사) |
| `verification_status` | string | 검증 상태 (`unverified`) |

<Note>
  생성 직후 상태는 `unverified`입니다. 이 자격증명으로 [세션을 생성](/api-reference/sessions)하면 로그인 성공 시 `verified`로 변경됩니다.
</Note>

## 자격증명 조회

메타데이터만 반환합니다. 비밀값(secrets)은 포함되지 않습니다.

```bash
GET /api/v1/credentials/{credential_id}
Authorization: Bearer {keyId}.{secret}
```

### 응답 — `200 OK`

```json
{
  "credential_id": "cred_57ea61f05c79",
  "account_id": "01KK45GW4R178J3BWRDDN630AP",
  "site": "wehago",
  "login_type": "password",
  "credential_fingerprint": "59d3a217953a9537",
  "verification_status": "verified",
  "verified_at": "2026-03-11T12:08:46Z",
  "created_at": "2026-03-11T11:30:22Z"
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `credential_id` | string | 자격증명 ID |
| `account_id` | string | 소유 계정 ID |
| `site` | string | 서비스 코드 |
| `login_type` | string | 로그인 방식 |
| `credential_fingerprint` | string | 자격증명 핑거프린트 (동일 자격증명 식별용) |
| `verification_status` | string | `unverified`, `verified`, `invalid` |
| `verified_at` | string \| null | 검증 완료 시각 (RFC3339) |
| `created_at` | string | 생성 시각 (RFC3339) |

---

## 자격증명 삭제

```bash
DELETE /api/v1/credentials/{credential_id}
Authorization: Bearer {keyId}.{secret}
```

### 응답 — `200 OK`

```json
{
  "status": "deleted"
}
```

<Warning>
  삭제된 자격증명으로 생성된 세션은 만료될 때까지 유효합니다.
  자격증명을 삭제해도 기존 세션이 즉시 무효화되지는 않습니다.
</Warning>

## 에러 코드

| HTTP | 설명 |
|------|------|
| `400` | 필수 필드 누락 또는 잘못된 요청 |
| `401` | 인증 실패 |
| `404` | 자격증명을 찾을 수 없음 (삭제되었거나 다른 계정 소유) |

---

## 보안

- **암호화 저장**: 비밀값은 Vault(HashiCorp)에 암호화되어 저장됩니다
- **비밀값 미노출**: 조회 API에 비밀값이 포함되지 않습니다
- **계정 격리**: 자격증명은 계정 단위로 격리됩니다. 다른 계정의 자격증명에 접근할 수 없습니다
- **핑거프린트**: 동일 자격증명 중복 로그인 방지에 사용됩니다 (비밀값 자체가 아닌 해시)

## 요금

자격증명 관리 API는 **무료**입니다.
