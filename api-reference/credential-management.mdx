# 자격증명 관리

외부 서비스(위하고, 홈택스 등)의 로그인 자격증명을 Vault에 안전하게 저장하고 관리합니다.
저장된 자격증명은 스크래핑 워크플로우에서 자동 로그인에 활용됩니다.

## 인증

모든 엔드포인트는 API 키 인증이 필요합니다.

```bash
Authorization: Bearer {keyId}.{secret}
# 또는
X-API-Key: {keyId}.{secret}
```

## 지원 프로바이더

| 프로바이더 | 값 | 설명 |
|-----------|------|------|
| 위하고 | `wehago` | 더존비즈온 위하고 |
| 홈택스 | `hometax` | 국세청 홈택스 |

## 자격증명 생성

```bash
POST /v2/credentials
Content-Type: application/json
Authorization: Bearer {keyId}.{secret}

{
  "provider": "wehago",
  "label": "본사 경리팀",
  "portal_id": "user@wehago.com",
  "password": "password123"
}
```

### 요청 필드

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `provider` | string | O | 프로바이더 (`wehago`, `hometax`) |
| `label` | string | | 식별용 라벨 |
| `portal_id` | string | O | 로그인 ID |
| `password` | string | O | 로그인 비밀번호 |

### 응답 — `201 Created`

```json
{
  "id": "cred-49fb8ddd-9d38-49fb-85dd-90176517640e",
  "provider": "wehago",
  "label": "본사 경리팀",
  "portal_id": "user@wehago.com",
  "created_at": "2026-03-09T05:37:41Z"
}
```

<ResponseField name="id" type="string">자격증명 ID (`cred-` 접두사 + UUID)</ResponseField>
<ResponseField name="provider" type="string">프로바이더</ResponseField>
<ResponseField name="label" type="string">식별용 라벨</ResponseField>
<ResponseField name="portal_id" type="string">로그인 ID</ResponseField>
<ResponseField name="created_at" type="string">생성 시각 (RFC3339)</ResponseField>

## 자격증명 목록 조회

```bash
GET /v2/credentials
Authorization: Bearer {keyId}.{secret}
```

### 쿼리 파라미터

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| `provider` | string | 특정 프로바이더만 필터링 (생략 시 전체) |

### 응답 — `200 OK`

```json
{
  "credentials": [
    {
      "id": "cred-49fb8ddd-9d38-49fb-85dd-90176517640e",
      "provider": "wehago",
      "label": "본사 경리팀",
      "portal_id": "user@wehago.com",
      "created_at": "2026-03-09T05:37:41Z"
    }
  ]
}
```

## 자격증명 수정

라벨 또는 비밀번호를 변경합니다. 전달된 필드만 업데이트됩니다.

```bash
PUT /v2/credentials/{id}
Content-Type: application/json
Authorization: Bearer {keyId}.{secret}

{
  "label": "본사 총무팀",
  "password": "newpassword456"
}
```

### 요청 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| `label` | string | 변경할 라벨 |
| `password` | string | 변경할 비밀번호 |

### 쿼리 파라미터

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| `provider` | string | 프로바이더 (기본값: `wehago`) |

### 응답 — `200 OK`

수정된 자격증명 객체를 반환합니다 (생성 응답과 동일 형식).

## 자격증명 삭제

```bash
DELETE /v2/credentials/{id}
Authorization: Bearer {keyId}.{secret}
```

### 쿼리 파라미터

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| `provider` | string | 프로바이더 (기본값: `wehago`) |

### 응답 — `204 No Content`

## 오류 코드

| HTTP | 코드 | 설명 |
|------|------|------|
| 400 | `INVALID_REQUEST` | 필수 필드 누락 또는 잘못된 요청 |
| 400 | `INVALID_PROVIDER` | 지원하지 않는 프로바이더 |
| 401 | `UNAUTHORIZED` | API 키 누락 또는 인증 실패 |
| 404 | `CREDENTIAL_NOT_FOUND` | 자격증명을 찾을 수 없음 |
| 500 | `INTERNAL_ERROR` | 서버 내부 오류 |

## 보안

- **암호화 저장**: 자격증명은 서버에 암호화되어 저장됩니다
- **비밀번호 미노출**: 조회 응답에 비밀번호가 포함되지 않습니다
- **계정 격리**: 자격증명은 계정 단위로 격리됩니다
- **TLS 전송**: 모든 통신은 HTTPS로 암호화됩니다

## 요금

자격증명 관리 API는 **무료**입니다.
