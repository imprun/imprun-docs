# 워크플로우

스크래핑과 로그인은 비동기 워크플로우로 실행됩니다.
이 API로 워크플로우 상태를 조회하고, 추가 인증에 응답하고, 필요 시 취소합니다.

<Note>
  워크플로우는 [스크래핑](/api-reference/scraping) 또는 [세션 로그인](/api-reference/sessions)에서 생성됩니다.
  이 API는 생성된 워크플로우를 관리하는 용도입니다.
</Note>

---

## 결과 조회

```bash
GET /api/v1/workflows/{workflow_id}
Authorization: Bearer {keyId}.{secret}
```

### 응답 — 처리 중

```json
{
  "status": "processing"
}
```

### 응답 — 완료

```json
{
  "status": "completed",
  "result": { ... },
  "session_id": "sess_LMCQe94B6mNaG1IcpNZ1n8X4"
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `status` | string | `completed` |
| `result` | object \| null | 스크래핑 결과 데이터 (로그인 전용 워크플로우는 `null`) |
| `session_id` | string | 세션 ID (세션이 포함된 경우) |

### 응답 — 추가 인증 필요

OTP, 캡차 등 사용자 입력이 필요한 경우 `challenge` 상태가 됩니다.

```json
{
  "status": "challenge",
  "challenge": {
    "id": "ch-otp-1773151766957",
    "type": "otp",
    "message": "OTP를 입력하세요",
    "expires_at": "2026-03-10T14:14:26Z"
  }
}
```

이 경우 [추가 인증 응답 API](#추가-인증-응답)로 사용자 입력을 전달합니다.

### 응답 — 실패

```json
{
  "status": "failed"
}
```

### 상태 값

| 상태 | 설명 | 다음 행동 |
|------|------|-----------|
| `processing` | 처리 중 | 2초 후 재조회 |
| `completed` | 완료 | `result` 필드 사용 |
| `failed` | 실패 | 원인 확인 후 재시도 |
| `challenge` | 추가 인증 필요 | challenge 응답 전달 |
| `cancelled` | 취소됨 | — |
| `terminated` | 강제 종료 | — |
| `timeout` | 시간 초과 | 재시도 |

## 추가 인증 응답

워크플로우가 `challenge` 상태일 때 사용자 입력을 전달합니다.

```bash
POST /api/v1/workflows/{workflow_id}/challenge
Content-Type: application/json
Authorization: Bearer {keyId}.{secret}
```

```json
{
  "challenge_id": "ch-otp-1773151766957",
  "data": {
    "otp": "123456"
  }
}
```

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `challenge_id` | string | O | `challenge.id` 값 |
| `data` | object | O | 인증 타입별 응답 데이터 |

### 인증 타입별 `data`

| 타입 | 설명 | data |
|------|------|------|
| `otp` | OTP 입력 | `{"otp": "123456"}` |
| `captcha` | 캡차 입력 | `{"captcha": "ABC123"}` |

### 응답 — `200 OK`

```json
{
  "status": "processing"
}
```

응답 후 결과 조회를 다시 시작하여 `completed`가 되었는지 확인하세요.

## 워크플로우 취소

```bash
DELETE /api/v1/workflows/{workflow_id}
Authorization: Bearer {keyId}.{secret}
```

### 응답 — `200 OK`

```json
{
  "status": "cancelled"
}
```

## 에러

| HTTP | 설명 |
|------|------|
| `401` | 인증 실패 |
| `404` | 워크플로우를 찾을 수 없음 |

## 폴링 가이드

워크플로우 결과를 조회할 때 적절한 간격으로 폴링합니다.

```python
import time
import requests

def wait_for_workflow(workflow_id: str, timeout: int = 300) -> dict:
    """워크플로우 완료 대기. challenge 발생 시 콜백 처리."""
    deadline = time.time() + timeout
    while time.time() < deadline:
        res = requests.get(
            f"{BASE}/api/v1/workflows/{workflow_id}",
            headers=HEADERS,
        ).json()

        match res["status"]:
            case "completed" | "failed":
                return res
            case "challenge":
                handle_challenge(workflow_id, res["challenge"])
            case _:
                time.sleep(2)

    raise TimeoutError(f"워크플로우 {workflow_id} 시간 초과")
```

<Warning>
  폴링 간격은 **2초 이상**을 권장합니다. 과도한 조회는 Rate Limit에 걸릴 수 있습니다.
</Warning>
